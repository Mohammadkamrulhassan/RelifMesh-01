\textbf{Jurisdiction Enforcement:} Every data query is filtered by \texttt{jurisdiction\_id}. A UP Official can only access their union's data; an Upazila Officer can access all unions under their Upazila.

\section{Data Privacy Design}
\textbf{PII Handling:} NID numbers are Sensitive PII --- visible only to authenticated users in the same jurisdiction, never in public responses. GPS coordinates are approximate in reports.

\textbf{Public Dashboard Rules:} Only aggregated counts per union (no per-household breakdown), no NID/name/GPS in public responses, pre-computed and cached endpoint.

\section{Input Validation \& Injection Prevention}
All inputs validated server-side with \texttt{express-validator} (trim, length checks, regex patterns). No raw SQL string concatenation --- parameterized queries throughout. React escapes output by default. Security headers via \texttt{helmet} middleware. Rate limiting: 200 requests per 15 minutes per IP.
