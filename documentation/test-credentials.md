## v2 OTP Auth Flow

v2 uses **Phone/OTP authentication**. The OTP is always randomly generated by the server. In development mode (`NODE_ENV=development`), the generated OTP is returned in the API response so you can use it immediately.

### How to Authenticate

```
1. POST /v2/auth/send-otp   →  Server generates 6-digit OTP, stores it (5 min TTL)
                                Returns: { message, otp }   (otp visible only in dev)

2. POST /v2/auth/verify-otp  →  Server verifies OTP hash
   { phone, otp }               If valid: returns { accessToken, refreshToken, user }
                                If invalid: returns 401 (max 5 attempts)

3. POST /v2/auth/refresh     →  Server rotates the refresh token
   { refreshToken }             Old token invalidated. Returns new { accessToken, refreshToken }
```

### Example (curl)

```bash
# Send OTP
curl -X POST http://localhost:5000/v2/auth/send-otp \
  -H "Content-Type: application/json" \
  -d '{"phone":"+8801700000008"}'

# Verify OTP
curl -X POST http://localhost:5000/v2/auth/verify-otp \
  -H "Content-Type: application/json" \
  -d '{"phone":"+8801700000008","otp":"<otp_from_above>"}'

# Use token
curl http://localhost:5000/v1/auth/profile \
  -H "Authorization: Bearer <accessToken>"

# Refresh
curl -X POST http://localhost:5000/v2/auth/refresh \
  -H "Content-Type: application/json" \
  -d '{"refreshToken":"<refreshToken>"}'
```

## OTP Configuration (Development)

| Setting | Value | Notes |
|---------|-------|-------|
| OTP length | 6 digits | `crypto.randomInt(100000, 999999)` |
| OTP storage | In-memory Map (fallback) or Redis | Auto-detects Redis, falls back to memory |
| OTP hashing | SHA-256 | Stored as hex digest |
| OTP TTL | 5 minutes (300s) | Auto-expires in Redis; Map cleanup on read |
| Rate limit | 3 OTP requests per 10 min per phone | Returns 429 with `retryAfter` |
| Max verify attempts | 5 per phone | Phone blocked for 30 min after 5 failures |
| Bypass | `NODE_ENV=development` → OTP returned in response body | Never returns OTP in production |
